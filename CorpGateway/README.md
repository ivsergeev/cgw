# CorpGateway

Tray application that exposes corporate JSON endpoints to an AI agent via a secure local HTTP API.

## Architecture

```
CLI Agent ──► localhost:9876/invoke ──► CorpGateway ──► Corporate endpoints
                  (bearer token)          (tray app)       (via browser session)
```

## Build & Run

**Prerequisites:** .NET 8 SDK

```bash
cd CorpGateway
dotnet restore
dotnet run
```

**Publish (Windows self-contained):**
```bash
dotnet publish -c Release -r win-x64 --self-contained -o ./publish
```

**Publish (macOS):**
```bash
dotnet publish -c Release -r osx-arm64 --self-contained -o ./publish
```

## Configuration

Config is stored in `%APPDATA%/CorpGateway/config.json` (Windows) or `~/.config/CorpGateway/config.json` (Linux/macOS).

```json
{
  "ApiPort": 9876,
  "ApiToken": "<auto-generated UUID>",
  "StartMinimized": false,
  "StartWithSystem": false
}
```

The API token is auto-generated on first run. Copy it from the tray menu or the main window.

## Local API Reference

All requests require: `Authorization: Bearer <token>`

### `GET /health`
```json
{ "status": "ok", "skills": 5 }
```

### `GET /skills`
Returns compact skill list for agent context (token-efficient format):
```
# Available skills
get_employee(id:int)  // Returns employee record by ID
search_orders(query:str, from:date?)  // Search orders
```

### `GET /skills/{name}/schema`
Returns full skill definition as JSON (for agent to fetch on demand).

### `POST /invoke`
Execute a skill:
```json
{
  "skill": "get_employee",
  "parameters": {
    "id": "42"
  }
}
```

Response: the JSON response from the corporate endpoint, proxied as-is.

## Skills format (`skills.json`)

```json
{
  "groups": [
    { "id": "hr", "name": "HR", "description": "HR system", "color": "#4F7EF7" }
  ],
  "skills": [
    {
      "id": "...",
      "name": "get_employee",
      "description": "Returns employee record by ID",
      "groupId": "hr",
      "url": "https://internal.corp/api/employees/{id}",
      "httpMethod": "GET",
      "parameters": [
        { "name": "id", "description": "Employee ID", "type": "Integer", "required": true }
      ],
      "cacheEnabled": true,
      "cacheTtlSeconds": 300
    }
  ]
}
```

## Agent integration (Claude Code / CLI)

Install Claude Code, then configure a skill file for the agent:

```markdown
# skills.md

You have access to a local CorpGateway API at http://localhost:9876.
Always include: Authorization: Bearer <token>

To call a skill:
POST /invoke  { "skill": "<name>", "parameters": { ... } }

Available skills (load full list with GET /skills):
get_employee(id:int)       // Employee record
search_orders(q:str)       // Search orders
```

## Security notes

- The API only listens on `localhost` — not accessible from the network
- Bearer token is required for every request
- Skills support GET, POST, PUT, PATCH and DELETE methods
- Response cache is in-memory, cleared on restart
- CDP connection uses existing browser session — no credentials are stored
